# 数据库说明

系统启动时会自动创建所需表，数据库表名和字段名使用英文 snake_case，保证 MySQL 与常见工具兼容。

## 核心表

- `admins`：管理员账号。
- `activation_code_batches`：激活码批次。
- `activation_codes`：激活码主表。
- `code_redemptions`：激活码兑换记录。
- `memberships`：QQ 号会员有效期。
- `operation_logs`：管理员操作日志。
- `activation_logs`：机器人激活和校验日志。

## 默认身份设计

机器人默认传：

- `subject_type = qq`
- `subject_id = QQ号`

后续如果改成微信、手机号或内部用户 ID，只需要换 `subject_type`，核心表结构不用改。

## 使用模式说明

- `single`：一次性卡，一张卡只能激活一次。
- `multi_single`：多人一次性卡，一张卡可以给多个 QQ 使用，但同一个 QQ 只能使用一次。
- `repeatable`：重复卡，不限制同一个 QQ 重复使用，适合测试码或特殊通用码。

## 状态说明

- `unused`：未使用。
- `partial`：部分使用，常见于多人一次性卡。
- `activated`：已达到最大使用次数。
- `disabled`：管理员禁用，不能继续激活。
- `deleted`：软删除，默认列表不展示。
