# 开发启动说明

## 1. 准备远程 MySQL

先在远程 MySQL 中创建数据库：

```sql
CREATE DATABASE activation_system CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

现在不需要在命令行填写数据库信息。第一次部署启动后，打开管理后台网页，会出现“系统初始化设置”页面，在网页中填写 MySQL、管理员和机器人密钥。

如果 MySQL 安装在同一台服务器宿主机上，MySQL 地址可以填写：

```text
127.0.0.1
```

当前 `docker-compose.dev.yml` 使用 `network_mode: host`，容器会和服务器宿主机共用网络，所以可以直接访问本机 MySQL。

## 2. Docker 热更新启动

```powershell
docker compose --progress plain -f docker-compose.dev.yml build
docker compose -f docker-compose.dev.yml up
```

- 管理后台：http://localhost:7790
- 后端接口：http://localhost:7790/api

Docker 部署模式只启动一个容器 `app`，前端静态文件由后端服务托管。

Dockerfile 已使用 `https://registry.npmmirror.com` 作为 npm 镜像源，依赖在镜像构建阶段安装，容器启动时不再重复执行 `npm install`。

网页填写的初始化配置会保存到：

```text
数据/系统配置.json
```

该目录会挂载进 Docker 容器，容器重建后配置不会丢失。

管理后台的 Vite host 和端口已经写在 `管理后台/package.json` 的 `dev` 脚本里：

```json
"dev": "vite --host 0.0.0.0 --port 7788"
```

## 服务器一键部署

在服务器上执行：

```bash
git clone https://github.com/TimShitPig/Activation-Code-Backend.git
cd Activation-Code-Backend
docker compose --progress plain -f docker-compose.dev.yml build
docker compose -f docker-compose.dev.yml up -d
```

然后打开：

```text
http://服务器IP:7790
```

第一次打开会进入“系统初始化设置”页面，在网页里填写 MySQL 和管理员信息。

如果需要修改端口、绑定地址、时区，或想用环境变量提前写入 MySQL 配置：

```bash
cp 部署配置/.env.example .env
```

编辑 `.env` 后再启动：

```bash
docker compose --progress plain -f docker-compose.dev.yml build
docker compose -f docker-compose.dev.yml up -d
```

不复制 `.env` 也可以直接启动，系统会使用默认端口 `7790`，并在第一次访问网页时进入初始化向导。

`--progress plain` 会把 Docker 构建进度改成普通文本输出，避免在部分终端里反复刷新同一批步骤。

常用命令：

```bash
docker compose -f docker-compose.dev.yml ps
docker compose -f docker-compose.dev.yml logs -f
docker compose -f docker-compose.dev.yml restart
docker compose -f docker-compose.dev.yml down
```

## 3. 本机不使用 Docker 启动

PowerShell 若拦截 `npm.ps1`，请使用 `npm.cmd`：

```powershell
npm.cmd install
npm.cmd run dev -w 后端服务
npm.cmd run dev -w 管理后台
```
